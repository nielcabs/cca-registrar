# Deployment Guide — Render.com (Free Tier)

Deploy the CCA Registrar System to a public URL in under 15 minutes.

## What You'll Need

- A GitHub account (free) — [https://github.com/signup](https://github.com/signup)
- A Render account (free, no credit card required) — [https://dashboard.render.com/register](https://dashboard.render.com/register)

---

## Step 1 — Push the Code to GitHub

### Option A: Using GitHub Desktop (easiest)

1. Download GitHub Desktop: [https://desktop.github.com](https://desktop.github.com)
2. Sign in with your GitHub account.
3. File → Add local repository → pick the `CASUGANATICS` folder.
4. If asked, click "Create a repository" and fill in:
  - Name: `cca-registrar-system`
  - Keep the repo **Private** (recommended — it contains seeded demo credentials).
5. Commit all files with a message like `Initial deployment`.
6. Click **Publish repository**.

### Option B: Using Command Line (if you know Git)

```powershell
cd c:\Users\Juniel\Downloads\CASUGANATICS
git init
git add .
git commit -m "Initial deployment"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/cca-registrar-system.git
git push -u origin main
```

---

## Step 2 — Deploy to Render

1. Go to [https://dashboard.render.com/](https://dashboard.render.com/) and sign in.
2. Click **New +** → **Web Service**.
3. Connect your GitHub account when prompted.
4. Pick the `cca-registrar-system` repository.
5. Render will auto-detect `render.yaml`. Confirm these settings:
  - **Name:** `cca-registrar` (or anything you like — this becomes the URL: `https://cca-registrar.onrender.com`)
  - **Region:** Singapore (closest to Philippines)
  - **Branch:** `main`
  - **Runtime:** Node
  - **Build Command:** `npm install`
  - **Start Command:** `node server.js`
  - **Plan:** **Free**
6. Click **Create Web Service**.

Wait ~5–8 minutes for the first build. You'll see logs like:

```
==> Downloading cache
==> Running build command 'npm install'
==> Running 'node server.js'
CCA Registrar System running at http://localhost:10000
```

When it says **Live**, open the URL shown at the top of the page.

---

## Step 3 — Warm It Up Before the Defense

Render's free tier **sleeps** after 15 minutes of inactivity. First request after sleep takes 30–60 seconds.

**Mitigation for the defense:**

1. Open your Render URL **5 minutes before the presentation** and log in once to wake it up.
2. Keep one browser tab open and occasionally click around during Q&A.
3. Optional: set up a free uptime ping at [https://uptimerobot.com](https://uptimerobot.com) (free, pings your URL every 5 minutes).

---

## Step 4 — Share the URL

You'll get a public HTTPS URL like:

```
https://cca-registrar.onrender.com
```

Send this to your panelists and adviser. It works on phones, tablets, and any computer.

## Known Limitations on Free Tier


| Limitation               | Impact                                     | Workaround                                  |
| ------------------------ | ------------------------------------------ | ------------------------------------------- |
| Sleeps after 15 min idle | 30–60s cold start                          | Warm up before demo                         |
| Ephemeral filesystem     | Uploaded receipts and DB reset on redeploy | Demo data auto-reseeds on every boot        |
| 512 MB RAM               | OCR may be slightly slower                 | Use clear, high-contrast receipts           |
| Slow CPU                 | Tesseract OCR takes 10–20s per image       | Expected — mention "processing" during demo |


**Tip:** If OCR is too slow on free tier during live demo, run the system locally on your laptop instead — OCR takes 2–5 seconds locally.

---

## Fallback: Run Locally (Most Reliable)

If the deployed version lags during the demo, switch to the local version:

1. On any laptop with Node.js 18+ installed:
  ```powershell
   cd CASUGANATICS
   npm install
   npm start
  ```
2. Open [http://localhost:3000](http://localhost:3000)

---

## Redeploying After Code Changes

After pushing new commits to GitHub, Render auto-deploys. You can also trigger a manual redeploy from the Render dashboard → **Manual Deploy** → **Deploy latest commit**.

---

## Environment Variables (already configured in `render.yaml`)


| Variable              | Purpose                       |
| --------------------- | ----------------------------- |
| `NODE_ENV=production` | Enables secure cookies        |
| `SESSION_SECRET`      | Auto-generated by Render      |
| `NODE_VERSION=20`     | Node.js version for the build |


You don't need to set these manually — they're defined in `render.yaml`.
